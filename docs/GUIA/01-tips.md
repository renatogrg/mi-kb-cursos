# Markdown Útil

Guía rápida de las características más utilizadas de Markdown para crear apuntes claros, consistentes y fáciles de mantener.

---

# Tips de Redacción

- ✅ Usa voz activa: **"Haz X"** en lugar de **"Se debe hacer X"**.
- ✅ Sé específico: **"Instala pip 26.1"** en lugar de **"Instala pip"**.
- ✅ Incluye datos cuando sea posible: **"+340 % de CTR"** en lugar de **"mejor resultado"**.
- ✅ Usa emojis con moderación para mejorar la lectura, no para decorar.
- ✅ Agrega enlaces internos a temas relacionados.
- ✅ Incluye ejemplos reales cuando ayuden a comprender el concepto.
- ✅ Mantén títulos y subtítulos descriptivos.
- ✅ Una idea principal por párrafo.
- ✅ Utiliza listas y tablas cuando faciliten la comprensión.

# Estructura Mínima

```markdown
# [Título]

## Concepto

[Explicación breve del tema.]

## Checklist

- [ ] Paso 1
- [ ] Paso 2

## Recursos

- [Documentación oficial](https://ejemplo.com)
```

---

# Encabezados

```markdown
# H1 - Título principal

## H2 - Sección

### H3 - Subsección

#### H4 - Apartado
```

---

# Texto

```markdown
**Negrita**

*Itálica*

***Negrita e itálica***

~~Texto tachado~~

`Código en línea`
```

Resultado

**Negrita**

*Itálica*

***Negrita e itálica***

~~Texto tachado~~

`Código en línea`

---

# Listas

## Lista no ordenada

```markdown
- Elemento 1
- Elemento 2
- Elemento 3
```

Resultado

- Elemento 1
- Elemento 2
- Elemento 3

---

## Lista ordenada

```markdown
1. Paso 1
2. Paso 2
3. Paso 3
```

Resultado

1. Paso 1
2. Paso 2
3. Paso 3

---

## Checklist

```markdown
- [ ] Pendiente
- [x] Completado
```

Resultado

- [ ] Pendiente
- [x] Completado

---

# Citas

```markdown
> Esta es una cita importante.
```

Resultado

> Esta es una cita importante.
